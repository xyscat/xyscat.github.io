模块机制

javascript 的发展应用：工具类库（解决浏览器兼容）、组件库（功能模块）、前端框架（功能模块组织）、前端应用（业务模块组织）。

CommonJS 规范

CommonJS 规范希望 JavaScript 能够在任何地方运行。

CommonJS 出发点

JavaScript 自身规范缺陷：

  • 没有模块系统：
  • 标准库较少：
  • 没有标准接口：
  • 缺乏包管理系统：JavaScript 应用中，无法自动加载和安装依赖。

CommonJS 规范的提出，主要是为了弥补当前 JavaScript 没有标准的缺陷，以达到像 Python、Ruby、Java 具备开发大型应用的基础能力。期望使用 CommonJS API 写出的应用可以具备跨宿主环境执行的能力，这样可利用 JavaScript 开发富客户端应用、其他：

  • 服务器端 JavaScript 应用程序
  • 命令行工具
  • 桌面图形界面应用程序
  • 混合应用

CommonJS 规范涵盖了 模块、二进制、buffer、字符集编码、I/O 流、进程环境、文件系统、套接字、单元测试、web 服务器网关接口、包管理……。

Node 借鉴了 CommonJS 的一些功能模块，在服务器端搭建了一个 JavaScript 运行环境。

CommonJS 的模块规范

  • 模块引用

var math = require('math');

CommonJS 规范中 require() 方法，接受模块标识，引入一个模块的 API 到当前上下文中。

  • 模块定义

在模块中，上下文提供 require() 方法引入外部模块。exports 对象用于导出当前模块的方法或者变量，是唯一导出的出口。在模块中，存在一个 module 对象，代表模块自身，exports 是 module 的属性。

在 Node 中，一个文件就是一个模块，将方法挂载在exports 对象上作为属性即可定义导出的方式：

  • 模块标识

模块标识就是传递给 require() 方法的参数，必须是字符串，或以. 开头的路径。可以没有后缀 .js。

Node 的模块实现

Node 对模块规范的实现进行了一些改善，规范中的 exports、require 和 module 看起来很简单，但具体的实现怎么样？

在 Node 中引入模块，经过 3 步：

  • 路径分析
  • 文件定位
  • 编译执行

Node 中，模块分为两种，核心模块、自定义文件模块。

  • 核心模块：在 Node 源代码的编译过程中，编译进了二级制执行文件。在 Node 进程启动时，部分核心模块直接加载进入内存中，引入这部分狠心模块，文件定位和编译执行可以省略，加载速度最快。
  • 自定义文件模块：在运行时动态加载，需要完整的路径分析、文件定位、编译执行过程，速度慢。

优先从缓存加载

前端浏览器会缓存静态脚本文件以提高性能；Node 对引入过的模块也都会进行缓存，以减少二次引入的开销。

浏览器缓存的是文件，Node 缓存的是编译和执行之后的对象。

无论核心模块和自定义文件模块，require() 方法都会对模块进行缓存，使其二次加载使用缓存优先。

核心模块的缓存检查优先于文件模块的缓存检查。

路径分析和文件定位

模块标识符分析

require() 方法接受一个表示符作为参数，在 Node 中，依据它进行模块查找。模块标识符在 Node 中主要分为：

  • 核心模块：http、fs、path等；
  • .开始相对路径文件模块；
  • /开始绝对路径文件模块；
  • 非路径形式的文件模块，自定义的 connect 模块；

核心模块：核心模块的优先级仅次于缓存加载。

路径形式的文件模块：在分析路径模块时，require() 方法会将路径转为真实路径，以其为索引，将编译执行后的结构存在缓存中，使二次加载更快。

自定义模块：自定义模块指的是非核心模块，不是路径形式的标识符。是一种特殊的文件模块，是一个文件或包的形式。这类模块的查找最费时。

模块路径：模块路径时 Node 在定位文件模块的具体文件是制定的查找策略，表现为一个路径组成的数组。

  • 创建 module_path.js 文件，内容为 console.log(module.paths);
  • 将其放在任意一个目录，执行 node module_path.js

模块路径的生成规则：

  • 当前文件目录下的 node_modules 目录
  • 父目录下的 node_modules 目录
  • 父目录的父目录下的 node_modules 目录
  • 沿路径向上逐级递归，知道根目录下的 node_modules 目录

这种生成方式与 JavaScript 的原型链或作用域链的查找方式相似。

在加载过程中，Node 会逐个尝试模块路径中的路径，知道找到目标文件为止。

当前文件的路径越深，模块查找耗时越多，自定义模块的加载速度是最慢的。
